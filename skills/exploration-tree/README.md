# Exploration Tree

> 通用探索树 — 树状探索记录框架，系统性求解复杂问题

## 这是什么

Exploration Tree 是一个结构化的问题求解框架。当面对复杂、多路径的问题时（调试、架构选型、性能优化等），AI Agent 容易陷入"一条路走到黑"或"随机尝试"的低效模式。

本 skill 提供**树状探索日志**，强制 Agent 采用 **"广度枚举 → 选易深入 → 失败回溯 → 再广度"** 的交替策略，确保：
- 不遗漏可能的解决路径
- 不在死胡同反复打转
- 每次失败都产出新信息，缩小搜索空间

## 为什么需要它

| 痛点 | Exploration Tree 的解决方案 |
|------|---------------------------|
| Agent 一条路走到黑，忽略其他可能 | 强制广度枚举：先列出所有候选路径再深入 |
| 失败后无记录，重复踩坑 | 树状日志持久化：每个节点记录状态、排除原因、时间戳 |
| 多条路径之间缺乏比较和排序 | 评估排序：按 (可行性 × 收益 / 成本) 选择下一个目标 |
| 长时间探索后迷失方向 | 层级耗尽触发反思：分析共同失败模式，生成新假设 |

## 核心工作流

```
广度枚举 → 评估排序 → 深度钻探 → 失败回溯 → 层级耗尽 → 反思 → 新广度层
    ↑                                                              │
    └──────────────────────────────────────────────────────────────┘
```

1. **广度枚举** — 列出当前层所有候选路径 + 难度评估，浅探测判断可行性
2. **评估排序** — 优先选难度低、收益高的目标
3. **深度钻探** — 选中目标后全力投入，允许展开子节点
4. **失败回溯** — 标记排除原因，立即选下一个
5. **层级耗尽** — 触发反思，生成新的广度层
6. **持久化** — 每次状态变更同步写入 `state.json`

## 文件布局

```

```
