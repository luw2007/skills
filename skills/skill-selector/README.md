# Skill Selector

> 通用 Skill 选择器 — 源码级深度分析 + 决策树，1-3 轮交互选定最合适的 skill

## 这是什么

当面对多个功能相近的 skill 不知选哪个时，Skill Selector 不是简单对比描述文字，而是**将候选 skill 源码下载到临时目录，基于 SKILL.md 全文 + 代码实现 + 配置文件做深度分析**，确保对比矩阵的每一项都有源码级可追溯依据。

## 与"简版对比"的区别

| | 简版对比 | Skill Selector |
|---|---------|---------------|
| 数据来源 | 搜索结果简述（1-2 句话） | 下载源码，读 SKILL.md + 代码 + 配置 |
| 分析深度 | 基于描述文字猜测 | 提取实际调用的命令、环境变量、核心逻辑 |
| 对比可靠性 | 可能误判 | 每项结论可追溯到源码行 |
| 交互方式 | 一次性列表 | 决策树逐步收窄，最多 3 轮 |

## 核心流程

```
输入: skill 候选（关键词/名称/路径）
  │
  ▼
Step 0: 搜索 + 下载候选源码到临时目录
  │
  ▼
Step 1: 深度提取 → 结构化摘要（SKILL.md + 代码 + 配置）
  │
  ▼
Step 2: 预构建完整决策树（含叶节点对比矩阵）
  │
  ▼
Step 3: 沿决策树交互 → 用户选择即时下探，零延迟
  │
  ▼
Step 4: 到达叶节点 → 输出对比矩阵 + 推荐
  │
  ▼
清理: 删除临时目录
```

**关键设计**：Step 0-2 一次性完成所有计算。Step 3 纯遍历，用户每次选择后立即呈现下一层，无需等待。

## 支持的输入格式

- **搜索关键词**：`argos` → 自动 `ai-skills find`
- **skill 名称列表**：`argos-log, argos-query`
- **安装命令**：`ai-skills add anthropics/skills --skill skill-creator`
- **本地路径**：直接指向 SKILL.md 所在目录

## 交互示例

```
用户: 帮我选一个查 argos 日志的 skill

→ 搜索命中 8 个候选，下载并深度分析
→ 决策树构建完成（2 层）：

├── 日志查询 (5)
│   ├── 国内 (3) → 对比矩阵已就绪
│   └── 海外 (2) → 对比矩阵已就绪
├── 综合诊断 (1) → 直接推荐
└── 报警管理 (1) → 直接推荐

→ 你的主要需求是？ [日志查询 / 综合诊断 / 报警管理]
→ 用户选"日志查询" → 即时下探
→ 你需要查询哪个区域？ [国内 / 海外]
→ 用户选"国内" → 输出对比矩阵 + 推荐
```

## 安装

```bash
npx skills add luw2007/skills --skill skill-selector
```
